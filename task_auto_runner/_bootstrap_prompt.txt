あなたは Claude Code として task_auto_runner の実装を行います。

## 必須参照ファイル（既に作成済み）
- task_auto_runner/SPEC.md （全仕様）
- task_auto_runner/TASKS.md （実装チェックリスト）
- task_auto_runner/CLAUDE.md （実装ルール）

まず上記3ファイルを読み込んで把握してください。

## 実装対象
- task_auto_runner/runner.py
- task_auto_runner/claude_invoker.py
- task_auto_runner/gate_runner.py
- task_auto_runner/notifier.py
- task_auto_runner/run_auto_runner.bat

## 既存依存（流用すること）
- gate_checker/gate_check.py（subprocessで呼ぶ）
- cost_guard.py の get_costs() 関数
- daily_report.py の push_message() 関数（importかurllib版に書き直し）
- config/.env から LINE_CHANNEL_ACCESS_TOKEN を読む

## 完了条件
1. python task_auto_runner/runner.py --dry-run が exit 0 で終了
2. テスト用ダミー指示書を pending_tasks/ に置いた状態で実走可能（実際の実行はしない）

## 注意
- Windows環境。全Pythonファイル冒頭に sys.stdout.reconfigure(encoding='utf-8', errors='replace')
- subprocess は shell=False 推奨
- 日本語パス対応: os.getcwd() で動的取得
- 既存の cost_log.jsonl のフォーマット（"ts"キー使用）に注意
- LINE 月次200通上限。失敗時は静かにログ出して継続

実装を開始してください。完了したら「IMPL_COMPLETE」とだけ最後に出力してください。
