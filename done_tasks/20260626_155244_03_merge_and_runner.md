### merge_policy.py
SPEC.md Section 6 参照。
```python
@dataclass
class MergeDecision:
    field: str
    action: str  # "update" | "skip_existing" | "skip_no_value" | "conflict"
    old_value: Any
    new_value: Any
    confidence: float
    source: str

def decide_merge(
    existing_props: dict,
    extracted: dict[str, Any],
) -> list[MergeDecision]:
    ...
```
- 空欄判定: None, "", whitespace → 空欄
- 単価0 → 空欄扱い
- 既存値あり → skip_existing (conflict report生成)
- confidence < 0.5 → skip_no_value

### update_runner.py
メインエントリーポイント。
```bash
python update_runner.py --dry-run     # レポートのみ
python update_runner.py --shadow-write # shadow report出力
python update_runner.py --apply       # Notion更新
python update_runner.py --apply --limit 10  # 10件のみ
```

処理フロー:
1. config/.env から NOTION_API_KEY 読込
2. Notion Engineer DB 全件fetch (pagination対応)
3. 各レコードについて:
   a. 人員情報原文 + 備考LINEメモ を結合
   b. engineer_text_parser で解析
   c. 6つの field_extractor を実行
   d. merge_policy で更新判定
4. --apply 時:
   a. pre_update_snapshot.json 保存
   b. Notion API PATCH で空欄のみ更新
   c. update_log.json に記録
5. レポート生成:
   a. summary_report (コンソール出力)
   b. detailed_report.json (ファイル出力)

### Notion API仕様メモ（Cursorが参照）
- エンドポイント: PATCH https://api.notion.com/v1/pages/{page_id}
- ヘッダー: Authorization: Bearer {key}, Notion-Version: 2022-06-28
- スキル更新 (multi_select): 
  ```json
  {"properties": {"スキル": {"multi_select": [{"name": "PHP"}, {"name": "Java"}]}}}
  ```
- 単価更新 (number):
  ```json
  {"properties": {"単価（万円）": {"number": 65}}}
  ```
- 最寄り駅更新 (rich_text):
  ```json
  {"properties": {"最寄り駅": {"rich_text": [{"text": {"content": "大宮駅"}}]}}}
  ```
- 経験年数更新 (number):
  ```json
  {"properties": {"経験年数": {"number": 11}}}
  ```
- 稼働可能日更新 (date):
  ```json
  {"properties": {"稼働可能日": {"date": {"start": "2026-07-01"}}}}
  ```
- Rate limit: 3 requests/second (sleep 0.35s between calls)
