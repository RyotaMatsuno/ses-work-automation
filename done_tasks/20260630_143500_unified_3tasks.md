## タスクC: LINE auto-register — スキル抽出 + サマリー/スキルシート紐付け

### 対象: ses_work/line_webhook/

### 背景
LINE経由でエンジニア登録する際の運用フロー:
1. 松野/岡本がサマリー（テキストメッセージ）を送る: 「28歳女性、Java5年、基本設計から...」
2. 続けてスキルシート（Excelファイル）を送る
3. この2つは別メッセージだが、同一エンジニアとして紐付けてNotionに登録する必要がある

現状の問題:
- サマリーとスキルシートが紐付かない
- スキルがNotionのmulti_selectに設定されない（備考にテキストが入るだけ）

### C-1: サマリー→スキルシート紐付けセッション管理

送信者ごとに「登録セッション」を管理:
- テキストメッセージ受信 → 人員サマリーと判定 → セッション開始(pending)
  - セッション有効期間: 30分
  - 保持情報: sender_id, timestamp, summary_text, engineer_page_id(Notion)
- ファイルメッセージ受信 → 同一送信者のpendingセッションを検索
  - あり: Excelを解析→既存Notionページを更新（スキル・単価・経験年数・Driveリンク）
  - なし: ファイル単体でも新規エンジニア登録（スキル抽出は実行）

人員サマリー判定ルール:
- 「歳」「男性」「女性」「経験」「年」「スキル」「即日」「稼働」等のキーワード
- 1行目に年齢・性別パターン（例: "28歳女性"）

### C-2: Excelスキルシート解析

- openpyxlでExcel読み込み（LINE Content APIでダウンロード→一時ファイル保存）
- シート内テキストからスキル名を抽出
- 既存 line_webhook/skill_extractor.py を活用・強化
- Notionの「スキル」(multi_select)に設定（既存オプションにマッチするもののみ）
  既存オプション: Java, Python, PHP, JavaScript, TypeScript, C#, Node.js, React, AWS,
  インフラ, PostgreSQL, Oracle, Vue.js, MySQL, Swift, Azure, Linux, Go, Ruby, Docker,
  MongoDB, Spring, SQL Server, Django, Angular, Kubernetes, Kotlin, GCP, C言語, HTML,
  CSS, Shell, Laravel, Git, PMO, Spring Boot, HTML/CSS, Excel, Word, SQL, OCI,
  Terraform, RHEL, JP1, Windows Server, Windows, PowerShell, VBA, Access, C++, C,
  Jenkins, Zabbix, NestJS, Struts, Salesforce, COBOL, M365, Perl, VB.NET, VMware

### C-3: 単価・経験年数の自動設定
- テキスト/Excel内の単価情報 → 「単価（万円）」に設定
- 「経験X年」「X年目」「X年半」→「経験年数」(number)に設定
- 単価なし時の推定: 上級SE 75-80万 / SE 65-70万 / 上級PG 55-60万 / PG 45-50万

### C-4: Excel→Google Drive保存
- LINE Content APIでExcelダウンロード → Google Driveにアップ（エンジニア名_日付.xlsx）
- NotionのDriveリンクURLに設定

### 完了条件C
- [ ] サマリー→スキルシートのセッション紐付けが動作
- [ ] Excelからスキル抽出→multi_selectに設定される
- [ ] 単価・経験年数が抽出/推定される
