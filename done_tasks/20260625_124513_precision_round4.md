## Phase 1: validate_skill スコアベース拒否強化

修正ファイル: `mail_pipeline/skill_extractor.py`

### 目的
文章断片がNotionに書き込まれるのを防ぐ。現在の単一ルール判定を
スコアベースに拡張し、複合条件で判断する。

### 既存のvalidate_skill()のポジティブパターン判定（Step 4）の前に追加:

```python
def _rejection_score(value: str) -> int:
    score = 0
    # 接続詞で始まる
    if re.match(r'^(および|または|もしくは|ならびに|かつ)', value):
        score += 2
    # 助詞3個以上（の/を/に/で/が/は/も/と/へ/より/から/まで）
    particles = len(re.findall(r'[のをにでがはもとへ](?=[^゠-ヿ])', value))
    if particles >= 3:
        score += 2
    # 15文字超 AND 辞書になし（aliasesは呼び出し元でチェック済み）
    if len(value) > 15:
        score += 2
    # 経験/業務/対応/提案/推進/管理 等の汎用ビジネスワード含む
    biz_words = re.findall(r'(経験|業務|対応|提案|推進|管理|支援|確認|作成|実施|調整)', value)
    if len(biz_words) >= 2:
        score += 1
    # "〜レベル" "〜程度" "〜以上" "〜相当" suffix
    if re.search(r'(レベル|程度|以上|相当|未満|年以上)$', value):
        score += 1
    return score
```

validate_skill()のStep 3（blacklist）とStep 4（ポジティブ）の間に挿入:

```python
    # Step 3.5: スコアベース拒否
    if _rejection_score(value) >= 3:
        return False, value
```

**重要**: 辞書にヒットした値（Step 1-2でaccept済み）はここに到達しないので安全。

## Phase 2: 単価再抽出（null化レコードの回復）

新規ファイル: `matching_v3/recover_prices.py`

### 目的
Round 3で単価異常としてnull化した案件の原文から、正しい月額単価を
ルールベースで再抽出し、高品質率を回復させる。

### 処理

```
python matching_v3/recover_prices.py           # dry-run
python matching_v3/recover_prices.py --execute  # 本番
```

1. Notion案件DB: ステータス=募集中 AND 単価=null AND 案件情報原文≠空 を取得
2. 原文から単価パターンを検索:

```python
PRICE_PATTERNS = [
    # "単価: 65万" "単価65万円" "月額60〜70万"
    r'(?:単価|月額|精算|想定単価|予算)[：:\s]*(\d{2,3})(?:\s*[~〜～]\s*\d{2,3})?\s*万',
    # "55-65万" "60～70万" (範囲の下限を取る)
    r'(\d{2,3})\s*[~〜～\-]\s*\d{2,3}\s*万',
    # "650,000円" "700000円" → 万円換算
    r'(\d{3,7})\s*円',
]
```

3. 抽出後バリデーション:
   - 20万 <= value <= 200万 のみ有効
   - 円建ての場合: value / 10000 で万円換算
   - 範囲指定の場合: 下限値を採用
4. dry-run: レポート（案件名、原文抜粋、抽出値）
5. --execute: Notionに書き込み

## Phase 3: 勤務地extractor 新規作成

新規ファイル: `mail_pipeline/location_extractor.py`

### 処理

```python
LOCATION_PATTERNS = [
    # "勤務地: 東京都港区" "勤務地：品川"
    r'(?:勤務地|就業場所|勤務場所|場所|エリア|勤務先|最寄り駅)[：:\s]*(.+?)(?:
|$)',
    # "【勤務地】新宿" 
    r'【(?:勤務地|場所|エリア)】\s*(.+?)(?:
|$)',
]

REMOTE_KEYWORDS = ['リモート', 'テレワーク', 'フルリモート', '在宅']

def extract_location(body: str) -> str | None:
    for pat in LOCATION_PATTERNS:
        m = re.search(pat, body)
        if m:
            loc = m.group(1).strip()
            # 不要な装飾除去
            loc = re.sub(r'[（(].*?[)）]', '', loc).strip()
            loc = re.sub(r'[\s　]+', ' ', loc).strip()
            if len(loc) < 2 or len(loc) > 30:
                continue
            return loc
    # リモートキーワード検索
    body_lower = body.lower()
    for kw in REMOTE_KEYWORDS:
        if kw in body_lower:
            return 'リモート'
    return None
```

### mail_pipeline統合
既存のstructurerまたはmail_pipeline.pyの案件処理フロー内で、
スキル抽出の隣でlocation_extractor.extract_location(body)を呼び、
結果をNotionの「勤務地」プロパティに書き込む。

**条件**: 既に勤務地が入っている場合は上書きしない。
