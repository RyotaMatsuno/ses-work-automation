# 01 gate_checker LINE通知絞り込み（松野承認済み 2026-07-06）

## 背景
gate_checkerが技術NG（ジョブズが自走対応すべきもの）まで松野へ生レビュー文をLINE送信し、
「gate ok」返信を要求している。判断分担ルール（松野確認は①使い方②コスト③品質の3条件のみ）と
チャネル分離方針（LINE=マッチング専用、判断要求はClaude.ai）に違反。月200通上限も浪費。

## 変更内容（gate_checker/gate_check.py の通知関数）
1. verdict=OK → **LINE通知を送らない**（ログのみ。results/へのJSON保存は現状維持）
2. verdict=NG → **1行のみ送信**:
   `[gate] {phase} NG: {対象ファイル名のみ} → ジョブズ対応中・返信不要`
   （レビュー本文・「〜okと送ってください」等の返信要求文言は全廃）
3. 松野判断が必要な案件（仕様NG等）はジョブズがClaude.aiチャネルで提起する運用のため、
   コード側からの松野判断要求は一切行わない
4. push_or_log()・LINE残通数チェックは現状維持

## テスト
- gate_checker/tests/ に通知分岐のユニットテスト追加（OK時: push呼び出しゼロ / NG時: 1行のみ）
- 既存テスト全パス
