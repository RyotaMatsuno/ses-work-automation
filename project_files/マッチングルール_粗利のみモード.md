# 【ナレッジ】粗利のみマッチ（スキルフィルタ除外）の運用ルール

作成日: 2026-06-15
種別: ナレッジ・設計メモ（実装タスクではない）

---

## 背景

通常のマッチングロジック（run_reverse_matching）は以下のフィルタを持つ:
1. スキルフィルタ: 必須スキルが1件も一致しない案件を除外
2. 粗利下限フィルタ: 粗利5万未満の案件を除外

一部の人員（例: PMO専門・コーディネーター系）は：
- スキルが汎用的すぎてマッチしない
- 単価が低いため必須スキルより粗利重視でマッチさせたい
という特性を持つ。

---

## 運用ルール（2026-06-15 松野確定）

### 通常モード（全人員デフォルト）
- スキルフィルタ: 有効
- 粗利下限フィルタ: 有効（5万以上）
- 粗利上限フィルタ: 撤廃済み（2026-06-15）

### 粗利のみマッチモード（個別指示時のみ）
- スキルフィルタ: 無効（必須スキル不一致でも通過）
- 粗利下限フィルタ: 有効（5万以上を維持）
- 発動条件: 松野から「PHは粗利だけ見てマッチして」等の個別指示があった場合

---

## 実装方法

webhook_server.py の run_reverse_matching() または
line_query.py の handle_line_query() でエンジニアごとのフラグを見て切り替える。

フラグの持ち方（2案）:
A案: Notionエンジニアレコードの「備考（LINEメモ）」に `#skill_skip` タグを追記
B案: run_reverse_matching_full() 呼び出し時に skill_skip=True を渡す

推奨: A案（Notionで管理できてコード変更不要）

### A案の実装イメージ

```python
def run_reverse_matching(engineer, projects):
    eng_skills = set(engineer.get("skills", []))
    eng_price = normalize_price(engineer.get("price", 0)) or 0
    # 備考に #skill_skip がある場合はスキルフィルタをスキップ
    note = engineer.get("note", "") or ""
    skip_skill_filter = "#skill_skip" in note

    matches = []
    for proj in projects:
        req_skills = set(proj.get("required_skills", []))
        ...
        # スキルフィルタ
        if not skip_skill_filter:
            if req_skills and not any(s in eng_skills for s in req_skills):
                continue
        ...
```
