最終更新: 2026-04-24

## 1. 提案対象の除外ルール

### 人材側の絶対除外
- 外国籍人材
- 地方人材（関東案件メインのため）
- 短期案件が続いている人材（定着しない → 早期離脱リスク）
- ブランクがある人材（スキル鮮度・稼働意欲に不安）
- 既往歴のある人材（案件側が敬遠 → 面談通過率低下）

### 案件側の絶対除外
- PM/コンサル案件（拡大が見えづらいため扱わない）

### エンジニアDBでの管理
- 「提案対象フラグ」カラム使用、デフォルトTrue、該当者はFalseに

## 2. 情報の鮮度ルール

| 対象 | 有効期限 | 理由 |
|---|---|---|
| 人材情報 | 3週間前まで | それ以前は状況変化の可能性が高い |
| 案件情報 | 4営業日前まで | それ以降はクローズ済みの可能性が高い（CEOは普段1-2日でクローズ） |
| 並行情報 | 当日確認分のみ | 日付が変わったら再確認必須 |

## 4. 単価・スキルマッチ判定ロジック

### 基本ルール
- 平均目標粗利: 7万円 / 最低粗利: 5万円
- 尚可スキルまで完全マッチなら案件予算+2万上振れ可能

### 判定フロー
- 必須スキルに×あり → 提案対象外
- 必須全○ + 尚可○率50%以上 → +2万上振れ可（粗利7万目標）
- 必須全○ + 尚可○率50%未満 → 案件予算内（粗利5万確保）

### 単価交渉ルール（2026-04-20 更新）
- 調整の余地は最大5万円まで → 一旦提案して交渉打診する方針
- SE帯（65〜70万）の人はあまり調整できない（元々下限に近い）
- 70万で書いてある人は大体65万まで調整可能
- 5万超の乖離がある案件は人員側に提案しない

## 5. 並行ステータスと提案可否

### 業界用語
- 「提案のみ」「並行なし」= 面談未設定（スコア加算なし）

### 並行スコア

| ステータス | スコア |
|---|---|
| 面談調整中 | 1.5 |
| 面談予定 | 2.0 |
| 結果待ち（1〜2日） | 2.5 |
| 結果待ち（3〜7日） | 2.0 |
| 結果待ち（8〜14日） | 1.5 |
| 結果待ち（15日超） | 1.0 |
| オファー中 | 5.0 |

- 合計5.0未満 → 提案OK
- 合計5.0以上 → 提案NG

## 6. 案件タイマー

| パターン | 締切 |
|---|---|
| 通常案件 | 受信から3時間 |
| 急募案件 | 受信から2時間 |
| 中長期案件（6ヶ月以上） | 受信から6時間 |
| 面談設定済み案件 | 締切なし（集まった人から随時提案） |

### 面談設定済み案件
- 識別: 面談日時が明記されている案件
- 意向確認時に面談対応可否を追加確認
- 面談対応可能な人材のみ提案
- 面談日時の1時間前まで追加提案継続

---

## 7. 提案数ルール
- 1案件あたり3名パッケージ推奨
- 型: 松竹梅 / 本命2+チャレンジ1 / スキル差別化
- 3名集まらなければ集まった人数で提案

---

## 8. 成約フロー
①意向確認 → ②単価交渉 → ③提案 → ④面談調整 → ⑤面談 → ⑥結果確認 → ⑦成約
- 全ステップで送信前に人間チェック必須
- リマインドは不要
- 面談通過率約3割、成約には3名以上の提案が必要

---

## 9. 注力企業ランキング
- 面談化率35% / 成約実績30% / 面談→成約率15% / 案件具体性10% / 案件頻度10%
- ランク: S → A → B → C → D

## 10. AI体制（マッチングAI + ダブルチェックAI）

### マッチング担当AI
- 候補者の高速抽出 + 提案文ドラフト生成
- 速度優先

### ダブルチェック担当AI
- 除外ルール違反チェック（外国籍/地方/短期連続/ブランク/既往歴）
- 並行スコア超過チェック
- 情報鮮度チェック（人材3週間超/案件4営業日超/並行当日以外）
- 敬語ミスチェック
- 固有名詞マスキング漏れチェック
- 単価計算チェック（粗利5万未満を検出）
- 出力: OK / NG + NG理由
