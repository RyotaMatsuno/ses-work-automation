# 岡本向け 仕組み説明

---

## 今回何を作ったか（全体像）

俺たち2人がLINEとClaudeだけ開いてれば仕事が回る仕組みを作った。
メールアプリもブラウザも基本開かなくていい。

---

## 仕組みの流れ

```
【案件が来たとき】

① 所属/案件からLINEまたはメールで案件が届く
     ↓
② AIが自動で判定（人材か案件か仕分け → NotionのDBに登録）
     ↓
③ AIがエンジニアDBから候補者を自動マッチング
     ↓
④ 提案文ドラフトを自動生成
     ↓
⑤ LINEまたはClaudeに結果が届く
     ↓
⑥ ダブルチェック（除外ルール・単価・並行・敬語・マスキング）
     ↓
⑦ 確認して「送信して」→ Claudeがメール送信 or 個人LINEで送信
```

---

## 何が自動で、何が手動か

| 工程 | 自動/手動 |
|---|---|
| 案件受信 → DB登録 | 自動 |
| マッチング | 自動 |
| 提案文ドラフト | 自動 |
| ダブルチェック | 半自動（Claudeがチェック → 人間が確認） |
| 送信 | 半自動（Claudeが送信 → 人間が「送信して」と言う） |
| 面談調整・成約交渉 | 手動 |

## 岡本の日常フロー

### メール確認
Claudeに「最新メール見せて」と送るだけ。

### メール送信
Claudeに「○○さんにこれ送って」と内容を伝える → 確認 → 「送信して」で完了。

### 提案文チェック
提案文をClaude貼り付け → 自動チェック → 修正版が出る → OKなら送信。

### LINE案件
公式LINEに案件が来る → 自動でマッチング → 結果が返ってくる → 確認して個人LINEで送信。

## ルール（最低限覚えておくこと）

### 提案しちゃダメな人材
- 外国籍
- 地方在住（関東以外）
- 短期案件が続いてる人
- ブランクがある人
- 既往歴がある人

### 単価ルール
- 最低粗利: 5万円（案件単価 - エンジニア単価）
- 目標粗利: 7万円
- 単価調整は最大5万円まで

### 並行ルール
- 並行スコア合計5.0以上 → 提案NG
- AIが自動判定するから基本心配しなくていい

### 絶対ルール
- 送信前に必ず人間が確認
- AIの判定は参考。最終判断は自分
