# SPEC.md - 提案パイプライン（propose_pipeline）

最終更新: 2026-05-27

---

## 1. 目的

`matching_v2/result.json` を読み込み、マッチング済み案件ごとに以下を自動実行する。

1. 提案テンプレート（テンプレート3）に基づく提案メール本文を生成
2. ses-mail経由でIMAPに下書き保存（送信はしない）
3. 処理結果をコンソールに出力

松野が「下書きを確認して送信ボタンを押す」だけで提案が完了する状態を作る。

---

## 2. 入力

- `ses_work/matching_v2/result.json`
- `ses_work/config/.env`（ANTHROPIC_API_KEY、IMAP/SMTP認証情報）

---

## 3. 対象案件の絞り込み

result.jsonの各案件から以下を満たすものを対象とする：

| 条件 | 処理 |
|---|---|
| `candidates` が1件以上ある | 対象 |
| `candidates` が0件 | スキップ |
| candidateの `price` が整数で100以下 → 万円単位とみなす（×10000換算） | 正規化 |
| candidateの `price` が100001以上 → 円単位とみなす（そのまま） | 正規化 |
| candidateの必須スキルに×がある（result未設定・△含む） | 除外 |
| 案件の `budget` が万円単位で5万未満粗利確保できない単価の候補のみの場合 | スキップ |

---

## 4. 提案メール生成ロジック

### 4-1. 候補者の選定

- 必須全◯の候補者のみ対象
- 粗利計算: `budget(円) - price(円) >= 50000` を満たす候補者のみ
  - budgetが万円単位（100以下）の場合は×10000換算
- 最大3名まで（粗利が高い順）
- 3名未満でも、いる分だけで提案

### 4-2. 担当者の判定

- `project_owner` が「松野」→ 送信元: r-matsuno@terra-ltd.co.jp
- `project_owner` が「岡本」→ 送信元: r-okamoto@terra-ltd.co.jp
- `project_owner` が空 → 送信元: sessales@terra-ltd.co.jp

### 4-3. サマリーアピール文の生成（テンプレート5ルール）

| 条件 | アピール文 |
|---|---|
| 必須全◯ + 尚可全◯ | 必須・尚可ともにマッチ度高い人員 |
| 必須全◯ + 尚可◯率50%以上 | 必須全て満たしており、尚可も経験あり |
| 必須全◯のみ（尚可なし or 50%未満） | 必須スキル全て満たし即稼働可能 |

### 4-4. メール件名

```
【ご提案】{案件名}
```

### 4-5. メール本文テンプレート

テンプレート3（案件側向けパッケージ提案文）を使用：

```
{案件担当者名}様（省略可、「ご担当者様」とする）

いつもお世話になっております。

案件ご紹介いただきありがとうございます。
下記{提案人数}名いかがでしょうか。
①に関しては{サマリー1}、②に関しては{サマリー2}...でございます。

ご検討いただけますと幸いです。

━━━━━━━━━━━━━━━━━━
① {氏名}様
 単価: {提案単価}万円
 稼働開始: {available_date or "即日"}
 並行: なし（確認要）
 [スキル詳細]
 必須: {スキル名}: ◯, ...
━━━━━━━━━━━━━━━━━━
（2名目以降同様）
━━━━━━━━━━━━━━━━━━

何卒よろしくお願いいたします。
```

---

## 5. 下書き保存

- IMAP APPEND コマンドで Drafts フォルダに保存
- IMAP接続情報: mail65.onamae.ne.jp:993（SSL）
- .envから読み込む変数:
  - `MATSUNO_EMAIL` / `MATSUNO_PASSWORD`
  - `OKAMOTO_EMAIL` / `OKAMOTO_PASSWORD`
  - `SESSALES_EMAIL` / `SESSALES_PASSWORD`
- 件名・本文・From・To（仮）を含むMIMEメッセージを作成してIMAPにAPPEND

---

## 6. 出力

コンソールに以下を出力：

```
[案件名] → 提案人数X名 → 下書き保存: 送信元アドレス (件名)
  ①氏名 単価XX万 粗利XX万
  ②氏名 ...
スキップ: [案件名] → 理由（候補なし / 粗利不足 / 全員必須×）
```

---

## 7. 実行方法

```
python propose_pipeline/propose.py
```

---

## 8. エラーハンドリング

| エラー | 対応 |
|---|---|
| IMAP接続失敗 | エラー表示してその案件をスキップ、次の案件へ |
| 下書き保存失敗 | エラー表示してスキップ |
| result.jsonが存在しない | エラーメッセージ表示して終了 |
