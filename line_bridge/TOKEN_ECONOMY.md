# トークン経済ルール（全人格・全システム共通）

最終更新: 2026-06-08

1. 各人格は自分のドメインの資料だけ読み込む。全プロジェクトファイルを全員に積まない。
   - ジョブズ：経営・分業・ルーティング・各人格からの要約
   - ジラード：判断マニュアルv3・提案テンプレート集v1
   - 渋沢：契約マスター・freee設定・払出ルール
2. 受け渡しは要約パケットのみ。会話全文・生の思考は渡さない／返さない。
3. 会話はタスク単位で短く保つ。終わらない長尺チャットを避け、作業キューで区切る。
4. 安定した資料・人格定義はprompt cachingで使い回す。
5. 上限はCostGuardで握る。日次/月次/バッチ。LLM経路は必ずCostGuard下。
6. ジョブズは専門家の生出力を読み返さない。返ってきた要約＋懸念点だけで統合・判断する。
