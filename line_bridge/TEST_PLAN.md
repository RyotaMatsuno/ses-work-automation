# TEST_PLAN.md ｜ LINE指示橋＋AI作業キュー

## 正常系
- 「今日の案件」→ matching_v3がreplyで秒返答
- 案件文を転送 → 即時マッチ → 候補をreply
- 「この案件、提案文まで重作業で」→ キューに種別=営業/担当=girardで登録 → ジラードがドラフト → 完了通知

## 異常系・エッジ
- 種別が曖昧なメッセージ → 1問確認replyが出る（無限ループしない）
- 送信を含む依頼（「メール送って」）→ 自動送信されず人間確認=要で停止
- 空メッセージ/スタンプ/画像のみ → 無視 or 定型reply（クラッシュしない）
- 同一案件の重複依頼 → 重複検知（task_id = SHA256(user_id + message_id + timestamp_min)[0:16]）
  - 同一task_idが既存キューに存在 → 新規登録せずに「既にキュー登録済みです」とreply

## 実データ形式
- 実際のLINE転送案件（日本語・改行・記号混在）でルーター判定が崩れない
- スキルシート添付（PDF/Excel）が来た場合の分岐が正しい

## CostGuard
- 営業重作業を連投 → バッチ/日次上限で停止する
- matching_v3呼び出しはAPIコスト0で計上される

## LINE制約
- 非同期完了通知がpush 200通/月を超えそうな場合の節約動作（ダイジェスト化/reply優先）を確認

## 合格条件
- 正常系3つが通る／異常系で自動送信・DB暴走・クラッシュが起きない／CostGuardが効く／push節約が働く
