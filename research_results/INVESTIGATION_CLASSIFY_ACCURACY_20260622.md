# Investigation: 分類精度改善（人員混入問題）
Date: 2026-06-22

## 発見
project分類の500件監査で158件(31.6%)が実は人員情報メールだった。

## 根本原因
1. classify_by_ruleの判定順序: PROJECT → ENGINEERの順だったため、
   「案件」「万」等のproject指標がある人員メールがproject確定してしまう
2. ENGINEER_PATTERNSに「人材」「要員」の角括弧パターンが不足
3. 「プロパー」の文脈判定がない

## 修正内容
1. classify_by_rule: ENGINEER判定をPROJECTより先に移動
2. ENGINEER_PATTERNS: 9パターン追加
   - 【...人材/要員/技術者/社員情報】角括弧パターン
   - 弊社/当社/自社 + プロパー/社員/所属
   - 注力/営業中 + 人材/要員
   - OW人材/Roots人材等の企業名付き
   - 地域+プロパー（福岡プロパー等）
3. 過剰パターン修正:
   - 単独の万パターン（~万, 万~万）を削除（案件に誤爆）
   - エンジニア紹介を弊社+エンジニア紹介に限定
4. PROJECT_OVERRIDE追加:
   - 「要員募集」「エンジニア募集」は案件側の用語として保護

## GPT壁打ち結果
- subject-firstの二段階判定を推奨
- body検出はスコア制で将来導入（今回はsubjectのみ）
- 「プロパー」は弊社/当社/自社付きは人員、顧客/先方付きは案件
- 「ご紹介させていただきます」等の汎用文句はトリガーにしない

## テスト結果
17/17 (100%) — 人員10件+案件7件の全件正解
