# SPEC.md - 返信自動解析システム

## 目的
所属企業からの意向確認メール返信を自動解析し、以下をNotionに自動登録する。
- 並行スコア（判断マニュアルv3 §5準拠）
- 必須/尚可スキルの○×
- 提案可否判定（スコア5.0未満かつ必須全○ならOK）

## 入力
ses-mail MCPで取得した返信メール本文（松野/sessalesアドレス）

## 解析項目

### 1. 並行ステータス抽出
メール本文から下記を正規表現＋Claude APIで抽出：
- 「面談調整中」→ スコア1.5
- 「面談予定」→ スコア2.0
- 「結果待ち」+ 日数判定 → スコア1.0〜2.5
- 「オファー中」→ スコア5.0
- 「並行なし」「なし」→ スコア0

複数件ある場合は合計スコアを計算。

### 2. スキル○×抽出
メール本文のフォーマット（提案テンプレート集v1 テンプレート1準拠）から抽出：
- 必須スキル：○/×
- 尚可スキル：○/×

### 3. 提案可否判定
- 並行スコア合計 < 5.0 AND 必須スキル全○ → 提案OK
- それ以外 → 提案NG（理由も記録）

## 出力（NotionエンジニアDB更新）
対象エンジニアのページに以下を更新：
- 並行スコア（number）
- スキル判定メモ（rich_text）
- 提案可否（select: OK/NG/要確認）
- 最終更新日（date）

## 実装方針
1. `reply_parser.py`：メイン処理
2. Claude API（claude-haiku-4-5-20251001）でメール本文を解析
3. 解析結果JSONをNotionに書き込み
4. `--dry-run`オプション対応（Notion書き込みをスキップ）
5. smoke test用のサンプルメール本文あり

## 完了条件
1. py_compile確認
2. python reply_parser/reply_parser.py --dry-run でサンプル解析が通ること
3. 並行スコア・スキル○×・提案可否が正しく計算されること
