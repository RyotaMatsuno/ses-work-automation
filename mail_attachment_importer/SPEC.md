# SPEC.md - メール添付スキルシート自動取り込みシステム v2

最終更新: 2026-05-13

---

## 1. 目的

TERRA共通アドレス（sessales@terra-ltd.co.jp）に届くスキルシートメールを
自動解析してNotionエンジニアDBに登録し、マッチング精度を向上させる。

---

## 2. 対応パターン

| パターン | 形式 | 内容 | 処理 |
|---|---|---|---|
| A | 添付ファイル（Excel/PDF/Word） | 1人分スキルシート | ファイルをバイナリ取得→テキスト変換 |
| B | Google SpreadsheetのURL（本文中） | 1人分スキルシート | Playwrightでページ取得→テキスト変換 |
| C | Google SpreadsheetのURL（本文中） | 複数人リスト | Playwrightでページ取得→複数人抽出 |

### スキップ条件
- ログインが必要なスプレッドシート（アクセス拒否 or ログイン画面） → スキップ・ログ記録
- 対応外の添付形式（画像・動画等） → スキップ
- 既処理UID → スキップ

---

## 3. 処理フロー

```
[定期実行 or 手動実行]
  ↓
① IMAP接続（sessalesアドレス）
  → 未処理メールをUID順に取得
  → processed_ids.jsonで処理済みUIDをスキップ
  ↓
② メール解析（パターン振り分け）
  → 本文にGoogle SpreadsheetのURL含む → パターンB/C
  → 添付ファイルあり（Excel/PDF/Word） → パターンA
  → 両方あれば両方処理
  ↓
③-A 添付ファイル取得・テキスト変換
  → Excel: openpyxlで全シート・全セルをテキスト化
  → PDF: pdfplumberでテキスト抽出
  → Word: python-docxで本文テキスト化
  ↓
③-B スプレッドシート取得・テキスト変換
  → URLからスプレッドシートIDを抽出
  → Playwright（jobz-command経由）でページ全体のテキストを取得
  → アクセス不可（ログイン画面）→ スキップ
  ↓
④ Claude APIで構造化データ抽出
  → 抽出項目: 氏名・単価・稼働可能日・経験年数・スキル一覧
  → 1ファイル/1シートに複数人いる場合は全員分抽出（パターンC対応）
  → スキルはNotionの既存選択肢にマッピング
  → 抽出結果はJSONで返させる
  ↓
⑤ Notion重複チェック
  → 氏名でエンジニアDBを検索
  → 同名が既存にあればスキップ（上書きしない）
  → 存在しなければ新規登録
  ↓
⑥ Notion登録
  → 稼働状況: 「稼働可能」
  → 備考に元メール件名・受信日・送信元を記録
  → 登録完了後にprocessed_ids.jsonにUIDを追記
```

---

## 4. 抽出項目とNotionカラムマッピング

| 抽出項目 | Notionカラム | 型 | 備考 |
|---|---|---|---|
| 氏名 | 名前 | title | 必須。取得できなければスキップ |
| 単価（希望） | 単価（万円） | number | 万円単位。「~」は上限値を使用 |
| 稼働可能日 | 稼働可能日 | date | 「即日」は登録日を使用 |
| 経験年数 | 経験年数 | number | 年単位（小数点以下切り捨て） |
| スキル | スキル | multi_select | 既存選択肢にマッピング |
| メモ | 備考（LINEメモ） | rich_text | 元メール件名・受信日・送信元 |

## 5. Claude API プロンプト設計

システムプロンプト:
```
あなたはSESエンジニアのスキルシートから情報を抽出するAIです。
以下のテキストから全エンジニアの情報を抽出し、JSON配列で返してください。
JSON以外のテキストは一切出力しないでください。

抽出するJSON形式:
[
  {
    "name": "氏名（フルネーム）",
    "price": 単価の数値（万円、不明はnull）,
    "available_date": "稼働可能日（YYYY-MM-DD形式、即日は今日の日付、不明はnull）",
    "experience_years": 経験年数の数値（不明はnull）,
    "skills": ["スキル1", "スキル2", ...]
  }
]

スキルは以下のリストから該当するものだけを選んでください:
Java, Python, PHP, JavaScript, TypeScript, C#, Node.js, React,
AWS, インフラ, PostgreSQL, Oracle, Vue.js, MySQL, Swift, Azure,
Linux, Go, Ruby, Docker, MongoDB, Spring
```

## 6. スプレッドシートURL抽出ロジック

対象URL形式:
- `https://docs.google.com/spreadsheets/d/{ID}/...`
- `https://docs.google.com/spreadsheets/d/{ID}`

Playwright処理:
1. URLにアクセス
2. ページ読み込み待機（3秒）
3. ログイン画面検出（"accounts.google.com"へのリダイレクト or "サインイン"テキスト）→ スキップ
4. 全テキストを取得してClaude APIへ渡す

---

## 7. 実行方式

- **手動実行**: `python importer.py` で即時実行
- **定期実行**: Windowsタスクスケジューラで1日1回実行（朝8時）
- 実行ログは `importer.log` に追記

---

## 8. エラーハンドリング方針

| エラー種別 | 対応 |
|---|---|
| IMAP接続失敗 | ログ記録してスクリプト終了 |
| 添付ファイル変換失敗 | そのファイルをスキップ・ログ記録して継続 |
| スプレッドシートアクセス不可 | スキップ・ログ記録して継続 |
| Claude API抽出失敗 | そのファイルをスキップ・ログ記録して継続 |
| Notion登録失敗 | ログ記録・リトライ1回・失敗してもUIDは記録しない |
| 重複検出 | スキップしてログ記録 |

---

## 9. 対象メールの絞り込み

- 対象フォルダ: INBOX
- 対象期間: 直近30日（初回実行時）/ 前回実行以降（2回目以降）
- 添付なし・URLなしメールはスキップ
- 添付形式: .xlsx / .xls / .pdf / .docx / .doc のみ対象
